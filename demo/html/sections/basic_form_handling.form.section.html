Description:

This is an example of all the pieces being used together to process a new user form.

Notes:

For this demo, html and js validation will not be used to showcase the validator responses.

Bools are my favorite part of all of this since PHP has smart conversions for it. yes/no, on/off, 1/0, 'true'/'false' all cleanly convert to boolean via filter_var().

It is highly recommended to set a default value for the property on the Struct for any inputs that don't always get sent over, like checkboxes. Otherwise PHP might start yelling at you about accessing uninitialized properties.

Examples:

New User Form:
#[Required]
#[Trim]
#[NotEmpty]
#[ValidatorFailureMessage(NotEmpty::class, 'Name is required')]
#[Required]
#[NotEmpty]
#[RegexMatch('/^p[a|@|4][s|5]{2}w[o|0]rd$/i')]
#[ValidatorFailureMessage(NotEmpty::class, 'Password is required')]
#[ValidatorFailureMessage(RegexMatch::class, 'Password must be "password"')]
#[Required]
#[NotEmpty]
#[FileUpload]
#[Required]
