Description:

Hydration happens when an array of data assigned into an object's public properties using the Struct's hydrate method or the constructor. This brings up the question, what if the data needs to be pre-processed before being assigned to the property? That is where the hydration attributes come in to play.

Hydration attributes run pre-processing on the value before it is assigned to the property. Either by transforming the value, cleaning it up, or running pre-validation checks.

Hydration attributes are processed in order from top to bottom.

Helpful Note: One of the more useful combinations is using Trim followed up by UseDefaultOnEmpty or UseValueOnEmpty.

Important Note: Data is automatically converted between types when possible. This can not be turned off (blame PHP and its decision on how to handle strict_types), but enhanced conversions can be disabled. Please reference the docs for more info.

Usage:

Hydrate an object's public properties with $_POST data:

$Obj->hydrate($_POST);

You can also hydrate the object via the constructor:

$Obj = new Obj($_POST);
