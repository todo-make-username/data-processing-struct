Description:

Here are some examples showing how a Struct's validate method is used to validate an object's public properties using various attributes.

Validation attributes are processed in order from top to bottom.

Custom messages for each validation attribute can be used instead of the default ones by including a ValidatorFailureMessage.

Helpful tip: One of the more useful combinations is using Trim followed up by NotEmpty.

Usage:

$Response = $Obj->validate();
echo $Response->success ? 'Success' : 'Fail';

// array[string]
$all_messages = $Response->getAllMessages();

// PropertyValidationResponse
$SomePropertyResponse = $Response->property_responses['some_property_name'];

// array[string]
$property_messages = $SomePropertyResponse->messages;

Examples:

#[NotEmpty]
#[Trim]
#[NotEmpty]
#[NotEmpty]
#[ValidatorFailureMessage(NotEmpty::class, 'This is my custom error message!')]
#[RegexMatch(pattern: '/^\d+[A-Za-z]+$/')]
