What is this library?

The Data Processing Struct is meant to help us eliminate a lot of the repetitiveness, deep nested code, and potential bugs that come with data processing and validation. As an extra bonus, it makes code extremely self documenting. You'll see what I mean with some examples in the other tabs.

It does all of that by moving all the logic into PHP attributes on an object's properties. Other than the provided helper traits, the struct class contains no actual logic, yet can process and validate an entire submitted form by itself. All thanks to attributes.

The perfect example is form processing. Instead of having scattered logic to process your forms. You can now make a single struct (or multiple structs) per form. The data is then processed and validated and assigned to the struct's public properties for you based on the attributes you give each property. All the logic is on the surface, and easy to see, no more 15 levels of methods to dig through to find out if you are trimming a string or not.

The best part is, all of this is fully extendable. I provide some attributes, but you can create your own custom ones and assign them to properties just like the ones I provide. As long as they extend the base classes, this library will pick them up and call them.

Demo Description:

Each tab contains a short description with usage examples and interactive demos of various attributes that are available to use from this library.

Things to note:

When using these attributes, it is highly recommended to use named arguments like in the examples. They are designed that way for our sanity. The exception being some of the tailor attributes which are wrappers for common PHP functions.

This demo has file uploads, the files are not actually uploaded anywhere. PHP should delete them from the tmp dir after the script finishes running.

Basic Struct Class Setup:

#[ClassAttribute1]
class Obj extends Struct
{
	#[Attribute1]
	#[Attribute2]
	public string $prop;
}
